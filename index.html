Operational Flow

How Omen Works

STEP 01

Identity Layer

Developers create an OmenBadge via zkLogin, permanently linking Web2 reputation to their on-chain identity. The badge is soulbound — immovable at the Move bytecode level. Minting requires a refundable 50 USDC staked deposit, burned on slash. Sybil-resistant by design.

STEP 02

Reputation Engine

On-chain behavioral data is aggregated and processed into a real-time trust score. Audit metadata is stored on Walrus with cryptographic proofs anchored to the Sui blockchain.

STEP 03

PTB Firewall Routing

The assert_trade_safety function returns (bool, u64) — a routing skip, not a hard revert. A false return routes the trade to an alternative pool. No transaction fails. The DEX routes around the risk, invisibly to the user.

STEP 04

Agent Accountability

Hardcoded cascade slash: a Level 1 agent slash burns 100% of agent stake + 25% of parent stake. Level 2 grandchild = 15% grandparent stake. Level 3 = no cascade. Third-party agents require an explicit LIABILITY_WAIVER flag or are treated as direct children.
